# Example Repository Contract

Use authority by owned question:

- `docs/semantic-root.md` owns definitions and invariants.
- `docs/current-state.md` owns the maintained capability summary, while current code and checks can disprove it.
- `docs/adr/**` owns accepted design decisions.
- `docs/product/**` owns product hypotheses and target experiences.
- `docs/reports/**` owns only the checks that each report says it executed.
- `docs/proof/current.json` owns accepted and candidate claim state.

A file, type, fixture, or ticket does not prove that a capability is shipped. A static review does not prove runtime behavior.
