# Changelog

## 0.2.0 · 2026-07-13

Added repository and corpus mode after a multi-file regression test exposed gaps that single-document examples did not cover.

- Added project-local authority discovery instead of assuming one global source order.
- Added Source Ledger, Coverage Contract, Term Registry, and Current-State Gate.
- Made provenance, snapshot freshness, overlays, manifests, historical reports, and dirty working trees explicit.
- Added hard distinctions between file existence, implementation, verification, claim acceptance, and product delivery.
- Added progressive-disclosure output shapes for newcomer orientation, README rewriting, companion guides, and consistency audits.
- Added four corpus-level entropy lenses: authority, provenance and freshness, coverage, and verification.
- Added `references/repository-corpus-mode.md`.
- Added four repository evaluations and a synthetic repository fixture containing stale entry copy, a product hypothesis, a static report, missing evidence, a dirty snapshot, and an unaccepted claim.
- Expanded trigger-boundary evaluations for repository snapshots, archives, file inventories, and search-only tasks.
- Updated the Chinese and English README files to document repository mode and its evidence boundaries.

## 0.1.0

Initial public release.

- Added the seven-step semantic decompression workflow.
- Added twelve single-source entropy lenses and six common decompression techniques.
- Added cross-domain examples and eight output-quality evaluations.
- Added trigger-boundary evaluations and bilingual documentation.
